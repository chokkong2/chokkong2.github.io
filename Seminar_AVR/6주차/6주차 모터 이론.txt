스위칭 레귤레이터
장점 : 발열 개선
단점 : 노이즈 증가, 회로 복잡, 비용문제

발열은 PCB의 신뢰성과 연관되는 문제라서 비용이 증가하더라도 꼭 해결해야하는 문제임

LM2576T-5
강압을 목적으로 사용됨
Output Voltage Version에 따라 저항 R2의 값이 달라짐.
여기서 사용되는 출력 전압은 5V이므로 R2를 3.1k옴으로 사용하면됨

저항값을 높이면 시간지연이 생겨서 리플이 생길 수 있음
저항값을 낮추면 과도 응답 특성을 개선할 수 있지만 전력손실의 증가로 발열이 증가하고 효율이 나빠짐

외부 인덕터를 사용할 때는 인덕턴스는 허용전류를 생각해서 선택해야함. 그래프 참고

스위치가 켜지고나서 꺼질 때 다이오드가 바로 OFF되지 않고 ON을 유지함 그 짧은 시간동안 큰 전류가 흐르게됨
=> 그래서 trr이 짧은 다이오드인 쇼트키 다이오드를 사용함 1A 사양과 3A 사양의 대표적인 쇼트키 다이오드 표가 있음
     LM2576T-5 데이터시트에 나와있는 1N5822는 3A 사양, 최대 40V까지.




L298N     : 모터 드라이버를 통해 토크제어, 속도제어, 전압제어, 전류제어, 위치제어 등을 정확하게 할 수 있고 모터 보호 기능 또한 가능함.
동작전압 : 5~46V까지
DC 전류는 하나의 출력 당 최대 2A까지
IN1, IN2, IN3, IN4 4개의 입력핀 (5,7,0,12번 핀)
OUT1~OUT4 4개의 출력핀 (모터 연결) (2,3,13,14번 핀)
EnA, EnB 라는 enble단자 2개. => EnA에 high를 걸어주면 입력 In1,In2를 제어, EnB에 high를 걸어주면 In3,In4 제어.
                                     => Enable 단자에 pwm 신호를 줘서 속도를 제어할 수 있음 (EnA, EnB가 0이면 모터가 멈춤)
전원핀 Vs : 모터에 정격전압을 직접적으로 가해주는 단자 (supply voltage)   ex) 모터가 24V 짜리면 Vs핀에 24V 인가해주면됨 (4번 핀)
         Vss : 5V를 인가해줘야함 (전원공급)      (9번 핀)
         GND : 8번핀
1번, 15번 핀 : SenseA, SenseB - 모터의 회전수(속도)를 센싱해서 AD컨버터 등으로 이용할 수 있는 단자
                  Rsa, Rsb 라는 저항을 연결해줌. 만약 속도가 필요없으면 저항을 연결하지않고 핀을 바로 접지에 연결함.
                  각 저항에 흐를 수 있는 최대 전류가 2A임
                  최대 2A의 전류에 대해서 0.5~1.0V의 전압강하를 제공하는 Sense 저항 Rs 사용. => 전압강하 0.5V, 전류2A면 Rsa = 0.25옴 사용
                                                                                                                   => 전압강하 1.0V, 전류 2A면 Rsa = 0.5옴 사용
                  이때 전력까지 생각을 해줘야함.  2A가 흐를때 0.25옴의 저항을 사용하면 전압강하 0.5V이고 전력을 계산해보면 1W 임.
                  따라서 2A가 흐를때 0.25옴의 저항을 사용한다면 최소 1와트짜리 저항을 사용해야함 (회로를 설계할때 저항의 허용 전력도 꼭 생각해줘야됨)
전원단자에 100nF(=0.1uF)의 바이패스 커패시터 연결함 (노이즈 제거 -> 안정적인 동작)
내부에 H-Bridge 회로가 전류 증폭 및 방향 제어 역할을 함.
출력핀과 모터를 연결할 때 다이오드 4개 사용. 역전류 보호용


PWM 모드에서 TOP 값을 OCRnA로 지정하여 동작하는 모드에서는 OCnA 핀을 PWM 출력으로 사용할 수 없음.
나는 회로를 구성할 때 모터의 Enable 단자를 OC1A와 OC1B로 연결했기 때문에 TOP 값을 OCR1A로 지정하면 안됨.
따라서 TOP값을 ICR1으로 지정하여 사용해야함. 그러면 OC1A~OC1C핀을 모두 PWM 출력으로 사용할 수 있음.

2. USART로 UP 보내면 속도 올리고 DOWN 보내면 속도 내리기
